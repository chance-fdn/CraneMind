# 垃圾储坑智能化管控系统需求文档

## 1. 概述

### 1.1 项目背景
开发一套垃圾储坑智能化管控系统，用于垃圾焚烧发电厂或固废处理中心的储坑管理。系统必须接入 AI，实现智能调度、异常检测、数据预测等功能。

### 1.2 系统目标
- 实现垃圾储坑的智能化管控
- 集成 AI 能力提升运营效率
- 提供实时监控和数据分析
- 支持三维可视化展示

## 2. 技术栈

### 2.1 前端技术栈
- **框架**: Vue 3 + TypeScript + Vite
- **状态管理**: Pinia
- **路由**: Vue Router
- **3D 可视化**: Three.js
- **图表库**: ECharts
- **视频集成**: HTML5 视频 + 流媒体播放器
- **实时通信**: WebSocket + 轮询（后备）

### 2.2 后端技术栈
- **运行时**: Node.js
- **框架**: Express
- **数据库**: PostgreSQL + Redis
- **身份验证**: JWT

### 2.3 AI 集成
- **大语言模型**: GPT-4/Claude/Llama3
- **图像识别**: 百度 OCR/GPT-4V

## 3. 功能需求

### 3.1 核心监控功能

#### Requirement 1: 大屏监控中心
**User Story:** 作为操作员，我希望实时监控垃圾储坑全景，以便及时了解现场情况。

**Acceptance Criteria:**
1. 实时展示垃圾储坑全景监控画面
2. 支持视频流或定时截图显示
3. 支持全屏展示功能
4. 画面切换延迟不超过2秒
5. 支持多摄像头画面切换

#### Requirement 2: 3D 模型展示
**User Story:** 作为操作员，我希望通过3D模型查看垃圾坑状态，以便更直观地了解堆料情况。

**Acceptance Criteria:**
1. 使用 Three.js 实现垃圾坑三维可视化模型
2. 支持旋转/缩放视角操作
3. 实时显示行车位置
4. 显示堆料高度（颜色热力图）
5. 模型加载时间不超过3秒

#### Requirement 3: 行车运行趋势图
**User Story:** 作为操作员，我希望查看行车运行趋势，以便分析设备运行状态。

**Acceptance Criteria:**
1. ECharts 折线图或面积图展示
2. 显示 1~3 号行车的运行状态
3. 包含速度/负载/位置数据
4. 支持时间范围筛选
5. 数据刷新间隔可配置

#### Requirement 4: 实时时间显示
**User Story:** 作为操作员，我希望看到准确的系统时间，以便记录操作时间点。

**Acceptance Criteria:**
1. 系统时间同步显示
2. 精确到秒
3. 自动更新，无需手动刷新
4. 显示格式：YYYY-MM-DD HH:mm:ss

#### Requirement 5: 全屏展示
**User Story:** 作为操作员，我希望一键全屏查看监控，以便获得更好的视觉效果。

**Acceptance Criteria:**
1. 提供全屏按钮
2. 支持一键切换全屏模式
3. ESC 键退出全屏
4. 全屏状态下保持所有功能可用

### 3.2 行车控制功能

#### Requirement 6: 行车状态监控
**User Story:** 作为操作员，我希望实时查看行车状态，以便了解设备运行情况。

**Acceptance Criteria:**
1. 显示 1 号、2 号、3 号行车实时状态
2. 状态包括：在线/离线/运行中/待机/故障
3. 显示当前位置坐标
4. 显示抓斗状态（开/关/移动中）
5. 状态更新延迟不超过1秒

#### Requirement 7: 行车职责配置
**User Story:** 作为管理员，我希望配置行车职责，以便优化作业分配。

**Acceptance Criteria:**
1. 支持配置每台行车的职责
2. 职责类型：投料/堆料/翻料/移料
3. 配置实时生效
4. 记录配置变更历史
5. 需要管理员权限

#### Requirement 8: 行车告警系统
**User Story:** 作为操作员，我希望及时收到行车告警，以便快速处理异常情况。

**Acceptance Criteria:**
1. 支持异常告警（过载、超限）
2. 支持位置校验偏差告警
3. 支持遛钩告警
4. 告警实时推送到前端
5. 告警记录写入数据库
6. 告警弹窗显示详细信息

#### Requirement 9: 自动/手动模式切换
**User Story:** 作为操作员，我希望切换行车控制模式，以便在不同情况下灵活操作。

**Acceptance Criteria:**
1. 支持自动/手动模式切换
2. 手动模式提供启停、急停按钮
3. 手动模式提供方向控制按钮
4. 模式切换需要权限验证
5. 记录模式切换日志

#### Requirement 10: 行车启停控制
**User Story:** 作为操作员，我希望控制行车的启停，以便进行作业操作。

**Acceptance Criteria:**
1. 提供启动、停止、急停按钮
2. 需要权限验证
3. 按钮状态实时反映行车状态
4. 操作记录写入日志
5. 急停优先级最高

#### Requirement 11: 任务管理
**User Story:** 作为操作员，我希望管理行车任务，以便控制作业流程。

**Acceptance Criteria:**
1. 显示当前任务列表
2. 支持取消任务
3. 统计各行车完成的任务数量
4. 统计任务执行时长
5. 任务状态实时更新

### 3.6 AI 智能功能

#### Requirement 29: 发酵数据智能分析
**User Story:** 作为操作员，我希望 AI 分析发酵数据，以便获得翻堆建议。

**Acceptance Criteria:**
1. 输入近期垃圾池内的温度、湿度、甲烷浓度时序数据
2. AI 输出预测未来 24 小时的发酵状态
3. 推荐最优翻堆时间
4. 输出自然语言分析结果
5. 输出结构化数据格式

#### Requirement 30: 行车告警智能诊断
**User Story:** 作为操作员，我希望 AI 诊断行车告警，以便快速处理故障。

**Acceptance Criteria:**
1. 当发生"遛钩告警"或"位置校验超差"时自动触发
2. 发送告警上下文（行车 ID、时间、传感器数据）
3. AI 返回可能的原因
4. 提供处理建议
5. 前端告警弹窗中新增"AI 诊断"按钮

#### Requirement 31: 大物体图像识别告警
**User Story:** 作为操作员，我希望 AI 识别大物体，以便及时处理异常物料。

**Acceptance Criteria:**
1. 从摄像头定期截取卸料口图片
2. 调用视觉 AI API 判断是否有超大物体
3. 识别建筑物料、家具等大物体
4. 生成告警记录到"大物告警记录表"
5. 实时推送到前端

#### Requirement 32: 行车调度任务优化建议
**User Story:** 作为管理员，我希望 AI 优化调度任务，以便提高作业效率。

**Acceptance Criteria:**
1. 基于当前垃圾库存生成建议
2. 基于各区域堆料高度生成建议
3. 基于投料口需求生成建议
4. AI 生成调度任务建议列表
5. 支持一键创建任务
6. 需要管理员权限

## 4. 非功能需求

### 4.1 性能需求
1. 页面加载时间不超过3秒
2. API 响应时间不超过1秒
3. WebSocket 消息延迟不超过500毫秒
4. 支持至少100个并发用户
5. 3D 模型帧率不低于30fps

### 4.2 安全需求
1. 所有 API 请求需要 JWT 验证
2. 密码加密存储
3. SQL 注入防护
4. XSS 防护
5. CSRF 防护

### 4.3 可用性需求
1. 系统可用性不低于99.5%
2. 支持故障自动恢复
3. 提供错误提示和日志
4. 支持数据备份和恢复

### 4.4 兼容性需求
1. 支持 Chrome、Firefox、Edge 浏览器
2. 适配大屏（1920x1080 及以上）
3. 适配普通 PC 显示器

## 5. 约束条件

### 5.1 技术约束
1. 前端必须使用 Vue 3 + TypeScript
2. 后端必须使用 Node.js + Express
3. 数据库必须使用 PostgreSQL + Redis
4. 3D 可视化必须使用 Three.js

### 5.2 业务约束
1. 行车控制部分的手动操作在前端模拟
2. 不要求真实 PLC 通信
3. 状态通过 WebSocket 更新
4. Three.js 三维模型先用立方体堆代替

### 5.3 开发约束
1. 所有代码必须有中文注释
2. 权限控制需在路由（前端）和 API（后端）双重校验
3. 优先保证核心功能完整可用
4. 重要功能提供接口占位
